# Project Context: Scrivener Document Management System

## 1. Flutter Mobile App
- **Path**: `c:\Users\jomar\Desktop\document_management_system`
- **Function**: The primary application for users with accounts. It handles document uploads, signature request creation, field placement, and internal document management. It features a premium, DocuSign-inspired UI for professional document workflows.

## 2. Flutter Web App (Guest Signing)
- **Path**: `C:\Users\jomar\Desktop\document_management_web_signing`
- **Function**: A frictionless, account-free web portal for "Guest Signers." It allows users to access documents via unique URL tokens, view PDF content using `pdfx`, and complete signature fields (Draw/Type) without logging in. It uses a responsive layout optimized for both desktop and mobile web browsers.

## 3. Express Backend
- **Path**: `C:\Users\jomar\Desktop\document-management-backend`
- **Function**: The core API service built with Node.js and Express. It manages the database (MongoDB/Firestore), handles authentication, performs PDF processing, generates secure signing tokens, and manages the lifecycle of signature requests.

## Architecture & Design
- **Theming**: Consistently uses a premium "Navy/Blue" palette across all platforms.
- **Folder Structure**: Follows a "Template" architecture based on `lib/Template/OperatingSystem/[Platform]/Page/` for modularity and scalability.
- **State Management**: Uses `GetX` for routing and reactive state management in both Flutter projects.
